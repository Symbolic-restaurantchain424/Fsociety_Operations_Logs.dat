## 🔍 Understanding the content

The logs contain several specific types of information. Each folder includes a text file explaining its purpose.

* **Active Directory records:** These files show how permissions work within a large network. You see patterns involving user accounts and group policies.
* **Threat Hunting reports:** These documents describe how teams find hidden intruders in a system. They track specific actions and patterns of behavior.
* **Red Team TTPs:** This stands for Tactics, Techniques, and Procedures. It outlines the logical steps taken during a security test.
* **EDR Evasion notes:** These files provide insight into how security software monitors activity and how that monitoring is bypassed during standard operations.
* **Forensics logs:** Use these for practice. They simulate a real system event log. You can try to reconstruct the timeline of an incident using these files.

## 🛠️ Troubleshooting common issues

If you encounter issues while opening the files, try these steps:

* **File blocked:** Sometimes Windows blocks files downloaded from the internet. Right-click the file, select "Properties", and check the "Unblock" box near the bottom.
* **Missing content:** If you see an empty folder, re-run the "Extract All" process. Sometimes files fail to copy correctly if the connection drops.
* **Large file errors:** Some logs are large text files. Use a professional text editor like Notepad++ if the standard Windows Notepad runs slowly.
* **Permission errors:** Move the folder to your user "Documents" folder. Avoid keeping these logs on your Desktop if you use cloud syncing services like OneDrive, as this can cause sync errors with large data sets.
